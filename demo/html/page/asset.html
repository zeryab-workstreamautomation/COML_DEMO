Assets are used in COML to specify resources such as other COML files, images, css etc. An asset is specified by a javacript object called an `assetId`. An assetId is a Javascript object with a name (the filename of the asset, including its extension) and a type. The type specifies an arrangement such as the subfolder where the asset is stored. It may optionaly store other keys as defined by a particular use of COML. var myPage={name:'for.html',type:'page'}; var myImage={name:'tiger.png',type:'image'}; // You can specify asset ids as a string with just the name part, internally COML functions will create the assetid. // In the above pages example, 'for.html' is converted by the pages component: var myPage=restoreAssetID('for.html'); // will get {name:'for.html',type:'page'} // When using inside element tags, you can use a stringified version of the assetId (JSON with " replaced by ') /* */ // and finally to encode an object asset ID to an string: var myassetid=stringifyAssetID({name:'for.html',type:'page'}); // returns "{'name':'for.html','type':'page'}" ## Asset Functions The two functions `stringifyAssetID` and `restoreAssetID` convert between a string and object representation of an asset. This is useful when specifying an asset as a string attribute on a COML component element.
